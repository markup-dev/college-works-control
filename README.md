## 🎯 О проекте

**College Works Control** — это веб-система для управления учебными заданиями и проверкой работ в учебном процессе.  
Платформа объединяет три роли (студент, преподаватель, администратор) в едином интерфейсе и автоматизирует ключевые процессы: выдачу заданий, отправку решений, оценивание и администрирование пользователей.  
Проект ориентирован на колледжи, кафедры и образовательные команды, которым важны прозрачность, контроль и удобство работы.

---

## ✅ Ключевые возможности

- ✅ **Ролевая модель доступа**: студент / преподаватель / администратор.
- 📦 **Управление заданиями**: создание, редактирование, удаление, прикрепление материалов.
- 🚀 **Отправка и проверка работ**: загрузка файлов, оценивание, возврат на доработку.
- ⚙️ **Админ-панель**: управление пользователями, группами, предметами и системными логами.
- 🎨 **Современный интерфейс**: адаптивные React-компоненты и удобные модальные окна.
- ✅ **Импорт данных**: массовая загрузка пользователей, групп и предметов.
- 📦 **API-first архитектура**: фронтенд и бэкенд разделены и масштабируются независимо.

## 🚀 Начало работы (Getting Started)

### Требования

- `Node.js` (рекомендуется `20+`)
- `npm` (рекомендуется `10+`)
- `PHP` (`8.2+`)
- `Composer` (`2+`)
- `MySQL` (`8+`)
- Git

### Установка

```bash
git clone https://github.com/markup-dev/college-works-control.git
cd college-works-control
```

#### 1) Настройка backend (Laravel API)

```bash
cd backend
composer install
cp .env.example .env
php artisan key:generate
```

Отредактируйте параметры БД в `backend/.env`:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=college_works_control
DB_USERNAME=root
DB_PASSWORD=
```

Примените миграции и сиды:

```bash
php artisan migrate --seed
```

Запустите API:

```bash
php artisan serve
```

#### 2) Настройка frontend (React)

```bash
cd ../frontend
npm install
```

Создайте файл `.env` в `frontend`:

```env
REACT_APP_API_URL=http://localhost:8000/api
```

Запустите клиентское приложение:

```bash
npm start
```

<details>
<summary>🔍 Дополнительно: быстрый запуск в двух терминалах</summary>

```bash
# Терминал 1
cd backend
php artisan serve
```

```bash
# Терминал 2
cd frontend
npm start
```

После запуска:
- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:8000/api`

## ⚙️ Использование

### Сценарий пользователя

1. Откройте страницу входа.
2. Выберите роль (`student`, `teacher`, `admin`).
3. Выполните вход с вашими учетными данными.
4. Работайте в соответствующем дашборде:
   - студент: просмотр заданий, отправка работ;
   - преподаватель: создание заданий, проверка и оценивание;
   - администратор: управление пользователями, группами, предметами, логами.

### Пример API-запроса (авторизация)

```bash
curl -X POST http://localhost:8000/api/login \
  -H "Content-Type: application/json" \
  -d "{\"login\":\"admin_sidorov\",\"password\":\"Password123\",\"role\":\"admin\"}"
```
